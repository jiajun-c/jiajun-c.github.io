CS144 lab1

[TOC]

先放一张经典的图… Do you see the lab1 work ?

The work

我们需要去实现一个流重载器，因为在进行数据的接收时都是无序的，需要进行重新的组合得到一个有序的数据。

The requirement

The keypoint is the capacity. 这也是TCP实现限流的一个重要的手段

capacity 意味着从被读入到没有被组合的位置的长度不能大于capcaity，所以我们需要去定义额外的数据结果去保存我们的未被组装的数据。

The implement

实现起来其实自由度还是挺高的，对于未被组装的数据，我们的存储方式也是很多的

  1. string 相当于是char数组保存所有的数据
  2. map<int, string> 保存未被组装的字符串以及其对应的起点
  3. priorty_queue 其实和前面的是差不多

论简单程度，肯定是第1个

1. 添加新的成员变量

使用unordered_map保存失配的字符位置，相当于vis数组，很暴力的啦。

2. push_substring

  1. 判断能否被读入，由于我们的capacity的限制，所以读入的数据的长度是有限的
  2. 有结束标记时判断合理性，当可以完全被读入的时候，此时可以打上eof标记
  3. 判断是否完全组装，如果完全组装且有结束标记，直接结束
  4. 对没有被组装的部分进行组装
  5. 重复一遍第三步

测试通过，就是慢了点。。
